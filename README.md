# Thoughts

A structured knowledge base of ideas, plans, research, and learnings generated
through AI-assisted development workflows. Documents are created using
thoughts-related skills in [Claude Code](https://claude.ai/code) and serve as a
persistent record of technical exploration, decision-making, and implementation
planning.

## Structure

```
thoughts/
  ideas/        # Problem statements and proposed solutions
  plans/        # Implementation plans with phases and success criteria
  research/     # Technical investigations and comparisons
  learnings/    # Reusable insights distilled from past work
```

### Ideas

Structured problem statements that identify a gap or opportunity, explain
motivation, and sketch a proposed solution. Ideas often evolve into plans once
the approach is validated.

### Plans

Step-by-step implementation plans derived from ideas or direct tasks. Plans
include current state analysis, phased implementation steps, success criteria,
and risk considerations.

### Research

Technical investigations comparing tools, approaches, or patterns. Research
documents capture findings with enough detail to inform future decisions without
repeating the investigation.

### Learnings

Concise, reusable insights extracted from completed work. Learnings capture
non-obvious details, gotchas, and patterns that are worth remembering across
projects.

## Naming Convention

Documents are date-prefixed for chronological context:

```
YYYY-MM-DD-<topic-slug>.md
```

Learnings are the exception -- they use topic-based names without dates since
they represent durable knowledge rather than point-in-time artifacts.
